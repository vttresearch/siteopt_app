# SiteOpt Assistant
## User-Focused Achievement Summary

**Purpose:** Help users prepare inputs faster and understand data before running energy system studies.

---

# 1. What Users Can Do Now

- Ask for input changes in plain language (for example: increase selected demand values).
- Ask questions about project data without manually searching files.
- Request visualizations directly from chat.
- See generated plots in the familiar **Data Editor → Assistant Plot** view.

---

# 2. End-User Experience

- Assistant is available directly inside the web interface.
- Works in the currently active project context.
- Responses are readable and structured (tables, lists, summaries).
- Charts are presented in the same area where users already inspect data.

---

# 3. Separation of Responsibilities

**Web Assistant = Input Preparation & Insight**
- Helps users inspect, update, and understand model input data.
- Supports quick visualization for decision making before runs.

**Spine Toolbox = Model Execution**
- Remains the execution environment for optimization/simulation runs.
- Produces model run results after inputs are finalized.

This keeps workflow clear: **prepare in web app, execute in Spine Toolbox, review results.**

# 7. Practical Value for Users

- Less time spent on repetitive manual edits.
- Faster confidence checks with quick plots and summaries.
- Lower risk of confusion from switching between many tools too early.
- Better collaboration: requests and outputs are easy to discuss.

---

# 8. Message to Stakeholders

- We improved usability without changing the core simulation responsibility.
- The solution supports current user workflow rather than replacing it.
- It creates a clear bridge between input preparation and formal model runs.
